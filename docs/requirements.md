# lazycwl 要件定義書

## 1. プロジェクト概要

### プロジェクト名

lazycwl（Lazy CloudWatch Logs）

### 目的

AWS CloudWatch Logsをターミナルから素早く閲覧・調査するためのTUIツールを提供する。

### 背景・課題

- AWSマネジメントコンソールのCloudWatch Logs画面は操作が煩雑で、障害調査時のログ閲覧に時間がかかる
- 従来のログファイルベースの障害調査では、Vimを使った検索・フィルター・抽出（不要行の除外→別ファイル出力）が効率的だった
- CloudWatch Logsに対しても、このVimベースのワークフローをそのまま適用したい

### 解決アプローチ

- yazi風のTUIでロググループ/ストリームを素早くブラウズし、選択したログを `$EDITOR`（Vim）で開く
- ナビゲーション層（TUI）と閲覧・分析層（Vim）の2層構造により、CloudWatch Logsをローカルのログファイルと同じ感覚で扱えるようにする

## 2. システム構成

### アーキテクチャ

2層構造で構成する:

```
┌─────────────────────────────────────────────────┐
│  ナビゲーション層（TUI）                          │
│  - yazi風ファイラーUI                             │
│  - ロググループ → ストリーム の階層ブラウズ          │
│  - インクリメンタル検索・フィルター                  │
│  - プレビューペイン                                │
├─────────────────────────────────────────────────┤
│  閲覧・分析層（$EDITOR）                          │
│  - 取得したログを一時ファイルに書き出し             │
│  - $EDITORで開いてユーザーが自由に操作             │
│  - 検索、フィルター、別ファイルへの抽出             │
└─────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────┐
│  AWS CloudWatch Logs API                         │
│  - DescribeLogGroups                             │
│  - DescribeLogStreams                            │
│  - GetLogEvents                                  │
└─────────────────────────────────────────────────┘
```

| 項目 | 技術 |

## 3. 機能要件

### F1: ロググループ一覧表示

- AWS CloudWatch Logs の `DescribeLogGroups` APIを使用し、アカウント内の全ロググループを取得する
- TUI上にリスト形式で表示する
- 表示項目: ロググループ名、保持期間、格納サイズ

### F2: ログストリーム一覧表示

- 選択されたロググループに対し `DescribeLogStreams` APIを使用し、ストリーム一覧を取得する
- 表示項目: ストリーム名、最終イベント時刻、格納サイズ
- デフォルトソート: 最終イベント時刻の降順（最新が上）

### F3: yazi風階層ナビゲーション

- 3階層構造: ロググループ → ログストリーム → ログ内容
- `l` または `Enter`: 現在の選択項目の階層を深く進む
- `h` または `Backspace`: 一つ上の階層に戻る
- yaziと同様に、親階層でのカーソル位置は復帰時に保持する

### F4: リスト内カーソル移動

- `j`: カーソルを一つ下に移動
- `k`: カーソルを一つ上に移動
- `g`: リストの先頭に移動
- `G`: リストの末尾に移動

### F5: $EDITOR連携（ログ閲覧）

- ログストリーム（またはF14で複数選択したストリーム）を選択し `l` または `Enter` で確定すると、以下を実行する:
  1. 対象ストリームのログイベントを `GetLogEvents` APIで全件取得する
  2. F10のフォーマットで一時ファイル（`/tmp/lazycwl-<timestamp>.log`）に書き出す
  3. `$EDITOR` 環境変数で指定されたエディタ（未設定時は `vim`）を起動し、一時ファイルを開く
  4. エディタ終了後、一時ファイルを削除し、TUI画面に復帰する
- エディタ内ではユーザーが自由に操作可能（検索、フィルター、別ファイルへの保存など）

### F6: インクリメンタル検索・フィルター

- `/` キーで検索モードに入る
- ロググループ一覧画面: ロググループ名で絞り込み
- ログストリーム一覧画面: ストリーム名で絞り込み
- インクリメンタルに（1文字入力ごとに）リストをフィルタリングする
- `Escape` で検索モードを解除し、フィルターをクリアする

### F8: CLI引数による直接指定

- 以下のCLI引数をサポートする:
  - `--group <log-group-name>`: ロググループを直接指定
  - `--stream <log-stream-name>`: ログストリームを直接指定（`--group` と併用必須）
  - `--profile <aws-profile>`: AWSプロファイルを指定
  - `--region <aws-region>`: AWSリージョンを指定
- `--group` と `--stream` の両方が指定された場合、TUIをスキップして直接$EDITORでログを開く
- `--group` のみ指定された場合、そのグループのストリーム一覧画面からTUIを開始する

### F9: ページネーション

- ロググループ、ログストリーム、ログイベントの各APIレスポンスに対し、ページネーションを処理する
- 初回は1ページ分を取得・表示し、リスト末尾に到達した時点で次のページを非同期に追加取得する
- ユーザーにはシームレスなスクロール体験を提供する

### F10: ログイベントのフォーマット

- エディタに渡すログは以下の形式で整形する:
  ```
  [2024-01-15T09:30:00.000Z] ログメッセージ本文
  ```
- タイムスタンプはISO 8601形式とする
- 複数ストリームをマージした場合は、タイムスタンプ順にソートする

### F11: AWSプロファイル選択

- `--profile` フラグでAWSプロファイルを指定可能
- 未指定時は `AWS_PROFILE` 環境変数 → デフォルトプロファイルの順にフォールバックする
- AWS SDK for Go v2の標準的な認証チェーン（環境変数、共有認証情報ファイル、IAMロール等）に従う

### F12: リージョン指定

- `--region` フラグでAWSリージョンを指定可能
- 未指定時は `AWS_REGION` / `AWS_DEFAULT_REGION` 環境変数 → プロファイル設定の順にフォールバックする

### F13: プレビューペイン

- yaziと同様に、画面右側にプレビューペインを表示する
- ロググループ一覧画面: 選択中グループの最新ストリーム名と最終イベント時刻をプレビュー
- ログストリーム一覧画面: 選択中ストリームの最新ログイベント数行をプレビュー
- プレビュー内容はカーソル移動に連動して非同期に更新する

### F14: 複数ストリーム選択

- ログストリーム一覧画面で `Space` キーにより複数ストリームを選択・選択解除する
- 選択済みストリームは視覚的にマーク表示する
- 複数選択状態で `l` / `Enter` を押すと、全選択ストリームのログをタイムスタンプ順にマージして1つの一時ファイルとしてエディタに渡す

### F15: ソート切り替え

- ログストリーム一覧画面で `s` キーを押すとソート順を切り替える
- ソート順: 最終イベント時刻順（降順） ↔ ストリーム名順（昇順）
- 現在のソート順をステータスバーに表示する

### F16: リアルタイム監視（tail）

- ログストリーム一覧画面で `f` キーを押すとtailモードに入る
- 単一ストリーム（カーソル位置）または複数選択ストリーム（F14）に対応する
- AWS CloudWatch Logs の `StartLiveTail` API（EventStreamベースのストリーミング）でリアルタイムにログイベントを受信する
- tailモードでは全画面レイアウト（カラム分割なし）でログを表示する
- 画面構成:
  - ヘッダー: ロググループ名とストリーム名を表示
  - 本文: ログイベントを `[ISO8601] message` 形式で表示（最大1000件保持、古いものから破棄）
  - ステータスバー: Live/Paused状態、イベント数、操作ヒントを表示
- tailモード中のキーバインド:
  - `j` / `↓`: 下にスクロール
  - `k` / `↑`: 上にスクロール（過去ログを遡れる）
  - `g`: 最上部にジャンプ
  - `G`: 最下部にジャンプ（自動スクロール再開）
  - `p`: 一時停止/再開トグル（一時停止中もバッファにイベントは溜まる）
  - `q` / `Escape`: tailモードを終了しストリーム一覧に戻る
- スクロール動作:
  - Live時: 最下部にいる場合は新しいイベント到着時に自動スクロール。上にスクロール済みの場合は位置を維持
  - Paused時: j/kで自由にスクロール。新しいイベントはバッファに溜まるが表示位置は動かない
  - `G` 押下で最下部に戻り自動スクロールを再開

## 4. 非機能要件

### NF1: AWS認証

- AWS SDK for Go v2の標準認証チェーンに従う
- 認証の優先順位: 環境変数 → 共有認証情報ファイル（`~/.aws/credentials`）→ IAMロール
- `--profile` フラグによるプロファイル切り替えをサポートする（F11参照）

### NF2: パフォーマンス

- API呼び出しは非同期で行い、TUIの操作をブロックしない
- プレビューペインの更新はカーソル移動後にデバウンス（200ms程度）してからAPI呼び出しを行う
- ログイベント取得中はTUI上にローディングインジケーターを表示する

### NF3: 配布形態

- `go build` でシングルバイナリを生成する
- Nix Flake経由でのインストールもサポートする（`flake.nix` にパッケージ定義を追加）

### NF4: エラーハンドリング

- AWS API呼び出しエラー（認証失敗、権限不足、リージョン不正等）はTUI下部のステータスバーにメッセージを表示する
- ネットワークエラー時はリトライ可能であることをユーザーに通知する
- エディタの起動失敗時はエラーメッセージを表示してTUIに復帰する

### NF5: 対応環境

- OS: macOS, Linux
- ターミナル: 256色以上対応のターミナルエミュレータ
- Go: 1.21以上

## 5. UX仕様

### 画面レイアウト

yaziと同様の3カラムレイアウトを採用する:

```
┌──────────────┬──────────────────┬─────────────────────┐
│ 親階層       │ 現在の階層        │ プレビュー           │
│ (ロググループ)│ (ログストリーム)   │ (最新ログイベント)    │
│              │                  │                     │
│ /aws/lambda  │ > stream-001  ← │ [09:30:00] START... │
│ /aws/ecs     │   stream-002     │ [09:30:01] Loading  │
│ /app/api  ←  │   stream-003     │ [09:30:02] Done     │
│              │                  │                     │
├──────────────┴──────────────────┴─────────────────────┤
│ [Status] Sort: time ↓ | Region: ap-ne-1              │
└───────────────────────────────────────────────────────┘
```

- 左カラム: 親階層（現在の階層がストリーム一覧なら、ロググループ一覧を表示）
- 中央カラム: 現在の階層のリスト（カーソル操作対象）
- 右カラム: プレビューペイン
- 下部: ステータスバー（ソート順、リージョン、エラーメッセージ）

### キーバインド一覧

| キー | 動作 |
|------|------|
| `j` / `↓` | カーソルを下に移動 |
| `k` / `↑` | カーソルを上に移動 |
| `l` / `Enter` / `→` | 階層を深く進む / ログをエディタで開く |
| `h` / `Backspace` / `←` | 階層を戻る |
| `g` | リスト先頭に移動 |
| `G` | リスト末尾に移動 |
| `/` | インクリメンタル検索モード |
| `Escape` | 検索解除 / モード解除 |
| `Space` | ストリームの複数選択トグル |
| `s` | ソート切り替え |
| `f` | リアルタイム監視（tailモード）開始 |
| `q` | アプリケーション終了 |

### エディタ連携フロー

```
TUI（ストリーム選択）
  ↓ l / Enter
ログ取得中（ローディング表示）
  ↓
一時ファイル書き出し
  ↓
$EDITOR 起動（TUIは一時停止）
  ↓
ユーザーがVimで作業（検索、フィルター、:w で別ファイル保存など）
  ↓ :q でエディタ終了
TUI に復帰（カーソル位置は保持）
```

## 6. 対象外（スコープ外）

以下は本プロジェクトのスコープに含めない:

- CloudWatch Logsへの書き込み・ログ送信
- CloudWatch Metrics / Alarms の表示・管理
- CloudWatch Logs Insights のクエリ実行
- ログの永続的なローカル保存・キャッシュ機構
- Windows対応
